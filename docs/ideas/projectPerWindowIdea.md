# Project-per-Window Architecture (VS Code Pattern)

> Status: P2 — 현재 드롭다운 유지, multi-window 지원 시 전환
> Created: 2026-04-12

## 배경

현재 단일 앱 인스턴스에서 프로젝트 드롭다운으로 전환하는 구조인데:
- ContextPack, PTY 세션, rawq 인덱스가 전부 프로젝트 종속
- 프로젝트 전환 비용이 큼 (인덱스 재로드, 맥락 변경)
- 실제 사용에서 프로젝트 간 빈번한 전환은 드묾

## 제안

VS Code처럼 **프로젝트 = 앱 윈도우 인스턴스**로 전환:
- 앱 시작 시 프로젝트 선택 (ProjectStartup — 이미 있음)
- 프로젝트 전환 = `tauri::Window::new`로 새 창
- 사이드바에서 프로젝트 드롭다운 제거 → 타이틀바에 프로젝트 이름만 표시
- 각 윈도우가 독립적인 DB 연결, PTY 세션, rawq 인덱스 보유

## 필요한 변경

1. **Tauri multi-window**: 윈도우별 프로젝트 키 바인딩
2. **DB 분리 또는 윈도우별 필터**: 현재 단일 DB → 프로젝트별 DB 또는 윈도우별 project_key 필터
3. **사이드바**: 프로젝트 드롭다운 제거, 타이틀바에 프로젝트명
4. **PTY/rawq**: 윈도우별 독립 세션

## 트레이드오프

| 장점 | 단점 |
|------|------|
| 맥락 손실 없음 | 메모리 사용량 증가 (윈도우 × ORT 모델) |
| VS Code 사용자에게 익숙 | multi-window Tauri 구현 복잡 |
| 프로젝트 간 간섭 없음 | 크로스 프로젝트 검색 불가 |
