# scaffold 사용자 customize 보존 + result task 차단

## 0. Context

### 0.1 외부 사용자 보고 (devbug, GitHub #254)

> "플랜 작성 중 마지막 task 에 모든 작업의 결과를 result 문서로 작성하라고 계획 작성하는 에이전트가 플랜을 짭니다. 이때 result 문서의 포맷까지 규칙에 들어가 있게 되구요. 이러고 플랜을 짜고 개발을 진행하면 이후 impl-complete 이 발행되고나서 result 문서를 자동으로 덮어쓰는거 같습니다. 리뷰어가 규칙을 보고 result 문서를 봤을 때 문서가 계속 포맷에 맞지 않게 작성되게 나와서 리뷰가 반복적으로 실패해서 해결을 할 수가 없습니다.
>
> 따라서 docs/agents/ 아래 문서에 result 문서에 대해 작성하는 task 만들지 말라고 명시적으로 박는 걸로 해결 했습니다. 하지만 tunaFlow 를 재실행 하면 해당 파일들이 다시 되돌려지는거 같습니다."

### 0.2 두 영역의 root cause

#### 영역 A — Architect prompt template 의 result task 자동 inject

`commands/project_tools.rs` 의 ARCHITECT_TEMPLATE (또는 plan-generation 흐름) 이 *마지막 task 에 result 문서 작성 명시* 를 default 로 inject. PR #211 (`resultMdContaminationFix`) 가 *reviewer 입력 차단* 만 처리, *plan generation 시 result task inject 자체는 차단 X*.

→ Architect 가 plan 작성할 때 마지막 task = "result.md 작성" 자동 추가 → impl-complete 후 syncResultReport 가 자동 덮어쓰기 → reviewer 가 정책 위반 (PR #212 의 "Never read result.md") 인지 못 하고 verdict fail 반복.

#### 영역 B — scaffold 의 docs/agents/*.md 덮어쓰기

`projects.rs:380-410` 의 scaffold 로직은 *§1 (Project Overview)* 만 manual edit detect 후 skip. **다른 section + docs/agents/{architect,developer,reviewer}.md 전체 영역은 보존 정책 미명시**.

→ 사용자가 architect.md 에 "result task 만들지 말 것" 추가해도 tunaFlow 재시작 시 scaffold 가 덮어쓰기.
